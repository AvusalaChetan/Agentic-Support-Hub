# Agentic Support Hub - Current Problems

1. **Rate limiting**
   - The Groq API can hit daily token limits, resulting in 429 errors and requiring users to wait or upgrade their tier.

2. **Fake/static map UI**
   - The map widget does not use a real map provider (Google Maps, Mapbox, etc.), so driver location is not shown on a real map.

3. **Hardcoded userId**
   - The app uses a fixed userId (USR-001) for all requests, so multi-user support is not implemented.

4. **Sample data only**
   - The app uses demo/sample data for orders and users; no real data or production database.

5. **No mobile responsiveness**
    - UI may not be fully responsive or optimized for mobile devices.

6. **No real-time updates for order status**
    - Order status and driver location are not updated in real time unless the user asks.

7. **No input validation**
    - User input is not validated for format or correctness before sending to backend.

8. **No loading states for widgets**
    - Widgets (map, refund, history) do not show loading indicators while waiting for backend responses.

9. **No error logging**
    - Errors are not logged to a central system for debugging or monitoring.

10. **No production deployment**
    - App is not deployed to a production environment; only runs locally.
